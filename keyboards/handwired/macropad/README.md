- **keyboard.json** — Tells QMK everything about your hardware: which pins are rows/columns,

  This is the only file that needs to change if you rewire anything.

- **rules.mk (keyboard level)** — Intentionally nearly empty. Hardware config lives in keyboard.json.

- **keymaps/vial/rules.mk** — Enables Vial. Must be at keymap level, NOT keyboard level.

- **keymaps/vial/config.h** — Sets the unique Vial keyboard ID and the security unlock key combo.

- **keymaps/vial/keymap.c** — Your starting keycodes. After flashing, use the Vial app to remap
  keys without reflashing. Think of this as just the factory default.

- **keymaps/vial/vial.json** — The visual layout that appears in the Vial GUI.
  Shows which key is where on screen. Compiled into the firmware itself.

## Matrix Wiring Guide

### How a Key Matrix Works

Instead of wiring every key directly to the MCU (which would need hundreds of pins),
keys are arranged in a grid. Each key sits at the intersection of one ROW wire and one COL wire.
When you press a key, it connects that row and column. The firmware scans each row,
activating it one at a time, and checks which columns are connected — this is how it knows
which key was pressed.

**Diodes** prevent "ghosting" — phantom keypresses that would occur if multiple keys are
pressed simultaneously without them.

### Diode Direction: ROW2COL

**ROW2COL** means current flows from ROW → through the diode → through the switch → to COL.

- The **cathode** (striped end / the end with the band) of each diode connects toward the **ROW wire**.
- The **anode** (plain end) connects toward one pin of the switch.
- The other switch pin connects to the **COL wire**.

In plain terms: the stripe on the diode points AWAY from the switch, toward the row.

### Row Pin Assignments

Each row wire on your PCB/hand-wire connects to this GPIO pin on the MCU:

  - Row 0: **GP5**
  - Row 1: **GP4**
  - Row 2: **GP3**

### Column Pin Assignments

Each column wire connects to this GPIO pin on the MCU:

  - Col 0: **GP6**
  - Col 1: **GP7**
  - Col 2: **GP8**

## How to Build the Firmware

### Option A: GitHub Actions (Recommended — no software to install)

1. Create a GitHub repository.
2. Place the contents of this zip into the repo root (so `.github/` and `keyboards/` are at the top level).
3. Push to GitHub.
4. Go to **Actions** → latest workflow run → wait for it to complete.

## How to Flash

### RP2040 (Raspberry Pi Pico)
1. Hold the **BOOTSEL** button on the Pico while plugging in USB.
2. A drive called **RPI-RP2** appears on your computer.
3. Drag the **.uf2** file onto that drive. It flashes and reboots automatically.

### ATmega32U4 (Pro Micro / Elite-C)
1. Open **QMK Toolbox**.
2. Load the **.hex** file.
3. Press the **Reset** button on the PCB (or short RST to GND twice quickly).
4. QMK Toolbox will detect the bootloader and flash automatically.

### STM32 (Proton C / WeAct)
1. Hold the **BOOT** button while plugging in USB (enters DFU mode).
2. Use **QMK Toolbox** or `dfu-util` to flash the **.bin** file.

## How to Use Vial

1. Flash your keyboard.
2. Download Vial from [https://get.vial.today](https://get.vial.today) or use the web version.
3. Plug in the keyboard. Vial should auto-detect it.
4. If prompted to **unlock**, hold the unlock combo: **top-left key + bottom-right key** simultaneously.
5. Click any key on the visual layout and press the new key you want to assign.
6. Changes save to the keyboard's EEPROM instantly — no reflashing needed.

## Modifying This Firmware

### Change a key's function permanently (in firmware)
Edit `keymaps/vial/keymap.c`, change the keycode, rebuild and reflash.
Full keycode reference: https://docs.qmk.fm/keycodes

### Change which pins are rows/columns
Edit `keyboard.json` → `matrix_pins` → `rows`/`cols`.
Make sure the new pins don't conflict with the TRRS serial pin (if split).

### Add a layer
In `keymap.c`, add a new `[1] = LAYOUT(...)` block with KC_TRNS for keys you want transparent.
Then assign a layer-switch key (e.g. MO(1)) on layer 0 to activate it.

### Change the USB device name
Edit `keyboard.json` → `keyboard_name`.

### Regenerate the Vial UID
If you fork this keyboard for a new design, run from the vial-qmk root:
`python3 util/vial_generate_keyboard_uid.py`
Then paste the result into `keymaps/vial/config.h`.
